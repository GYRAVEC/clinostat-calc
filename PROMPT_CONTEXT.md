# 2軸クリノスタット シミュレータ開発 — コンテキスト

## プロジェクトの背景と動機

私は GYRAVEC で 2軸クリノスタットによる **部分重力下でのバクテリア培養** 実験を進めています。目的は火星 (0.378g) や月 (0.166g) のような地球と異なる重力環境を地上で再現し、24-48時間の培養を通じて細胞挙動を観察することです。

特に重要な制約として、**懸濁状態を維持して沈殿させない** ことが求められます。なぜなら、火星 analog として現実の沈殿挙動を再現する研究もあるが、私のメイン用途は「部分重力下での細胞代謝・遺伝子発現を懸濁状態で観察する」ものだからです。

## 装置仕様 (ハードウェア)

| 項目 | 値 |
|------|----|
| 内側フレーム回転半径 | 8.5 cm |
| 外側フレーム回転半径 | 14.5 cm |
| 内軸と外軸の関係 | 互いに直交 |
| 外軸の鉛直からの傾斜角 ψ | 機械的に可変 (0° で鉛直、90° で水平) |
| サンプル配置 | 用途に応じて: 2軸交点 (中央) / 内側フレーム 4角 (r=8.5cm) / 任意 r |
| モーター | 外軸・内軸ともに独立速度制御、0.05〜200 RPM |
| サンプルチャンバ半径 | 5 mm (細菌培養液) |
| 培養対象 | バクテリア (自由沈降速度 ~1 μm/秒、生物応答時間 ~10 分) |

## 4つの運用モード (これまでに導出済み)

サンプル本体系で細胞が感じる時間平均重力 ⟨g⟩ をどう制御するかで 4 モードに分けています。各モードの導出と妥当性は GitHub に公開した [THEORY.md](https://github.com/GYRAVEC/clinostat-calc/blob/main/THEORY.md) に詳述しているので最低限読んでください。

### (1) Tilt モード (推奨、サンプル中央配置)

- 外軸を ψ = arccos(g_target) 傾ける
- 外側 ω_out 高速 (5 RPM)、内側 ω_in 超低速 (0.05 RPM = 20分で1周)
- 細胞は短時間平均 |⟨g⟩| = g₀·cos ψ = g_target を知覚
- 長時間平均 ⟨g⟩ = 0 (内側ドリフトで方向が一周するため)
- **達成可能: 0-1g 連続、沈殿なし**
- 中央配置のためフレーム半径 8.5/14.5 cm は構造寸法としてのみ意味を持ち、計算式に登場しない

### (2) Centrifuge モード (サンプル 4角配置)

- 内側遠心力 ω_in² × r で g_target を負荷 (火星なら 63 RPM @ r=8.5cm)
- 外側は地球重力打ち消し用のクリノローテーション (5 RPM)
- サンプル系で遠心力ベクトルは **body 固定方向** → §11 で証明したとおり回転を合成しても消せない → **必ず沈殿する**
- 火星表面で実際に起きる沈降挙動を再現するため、Mars analog 実験には適切

### (3) Hybrid モード (任意のサンプル位置 r)

- サンプルが軸交点から少しオフセット (例: 2cm) している場合の処理
- 重力 = √((g₀·cos ψ)² + (ω_in²·r)²) の直交合成
- 低速 ω_in (ドリフト用) では遠心寄与が極小 (r=2cm で 5.6×10⁻⁸ g)
- 実用上は tilt モードと等価。サンプル配置の柔軟性を確保

### (4) Switching モード (中央配置、傾斜なし運用)

- 装置を傾けられない場合の代替
- Mode A: 1:1 同期 + 微小オフセット (0.5g) と Mode B: 黄金比 (0g) を時間配分
- τ_A = 2·g_target で混合 → 連続的に g_target を作る
- 上限 0.5g (1:1 同期モードの数学的限界 §4)

### モード選択判断

| 実験目的 | 推奨 |
|---------|------|
| 部分重力下で懸濁培養 (メイン用途) | **Tilt** |
| 火星表面環境再現 (Mars analog) | Centrifuge |
| サンプル中央に置けない場合 | Hybrid |
| 傾斜マウントを作れない & 0.5g 以下で済む | Switching |

詳細は THEORY.md §14 の「従来手法との比較となぜ Tilt が優位か」を参照。

## なぜシミュレーションが必要か

ここまでは **時間平均** の解析理論ですが、実際に細胞が「**瞬間瞬間**」どんな重力を感じているか、48時間培養で細胞分布がどう発展するかは、time-series で見ないと分かりません。具体的には:

1. **時間平均理論の検証**: TC1〜TC8 (後述) で解析式 g_target × g₀ と一致するか
2. **過渡応答の可視化**: モード切替時 (switching) の細胞応答
3. **数値ドリフト確認**: 48時間にわたって誤差が累積しないか
4. **細胞分布予測**: Brown 運動 + Stokes 沈降を含む実際の細胞分布
5. **論文/プレゼン figure 作成**: 出版品質の図とアニメーション

## 作ってほしいシミュレータ

サンプル本体系で時刻 t における重力ベクトル `g_sample(t)` を、地球重力と内外遠心力をすべて含む完全モデルで時系列シミュレーションするツール。

### 必須機能

- 4 モード (tilt/centrifuge/hybrid/switching) + カスタムモード対応
- 時間離散化での高精度数値積分 (RK4 既定)
- 細胞 N 個の軌道シミュレーション (Stokes 沈降 + 任意で Brown 運動)
- 出版品質プロット 6 パネル + 3D アニメーション (オプション MP4)
- CSV/JSON 出力
- 検証テスト TC1-TC8 の自動実行

### 物理モデルの本質

地球重力は無視しません。サンプル系での合成重力は:

```
g_sample(t) = R_total⁻¹(t) · [-g₀·ẑ_lab + F_centrifugal_inner + F_centrifugal_outer]
R_total(t) = R_outer(α(t)) · R_inner(β(t))
α(t) = ω_out · t,   β(t) = ω_in · t   (モード依存で時間プロファイルあり)
```

回転の合成は `scipy.spatial.transform.Rotation` を必ず使ってください (自前で書くと SO(3) からドリフトします)。

### 推奨スタック

- Python 3.11+
- numpy (ベクトル演算)
- scipy.spatial.transform.Rotation (回転計算の核)
- matplotlib (プロット + アニメ)
- numba (細胞ループ JIT、24時間を 1分以内に)
- pytest (検証テスト)
- ffmpeg バイナリ (MP4 エンコード)

## 完了の定義

1. 検証テスト TC1〜TC8 すべて pass
2. 解析理論値 (THEORY.md §6-§13) と 1% 以内一致 (短期平均)、1e-5g 以内 (長期平均)
3. 48時間シミュレーションが 60秒以内
4. プロットが論文 figure 水準 (300 dpi、軸ラベル、凡例、grid、colorblind-safe)
5. pytest 全 pass、カバレッジ 80% 以上

## カスタマイズすべき箇所

実情に応じて変えるべき値:

- 培養生物 (バクテリア → 哺乳類細胞、酵母など): v_sed, D, T_bio, chamber size
- 培養時間の最大値 (24-48h → 1週間など)

- モーター速度上限 (200 RPM 以外なら)
- 完了基準の厳しさ (高速 PC なら 48h を 30 秒以内、など)
